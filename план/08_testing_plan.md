# План тестирования

## Обзор

Детальный план тестирования игры на всех этапах разработки.

---

## Типы тестирования

### 1. Unit тесты
Тестирование отдельных функций и классов.

### 2. Integration тесты
Тестирование взаимодействия между системами.

### 3. Functional тесты
Тестирование функциональности игры.

### 4. Performance тесты
Тестирование производительности.

### 5. Cross-browser тесты
Тестирование на разных браузерах.

### 6. Device тесты
Тестирование на разных устройствах.

## Этап 3: Тестирование игровой логики

### Коллизии
- [ ] Столкновение с врагом
- [ ] Столкновение с препятствием
- [ ] Сбор коллекции
- [ ] Нет ложных срабатываний
- [ ] Производительность приемлемая

### Спавн сущностей
- [ ] Сущности спавнятся в правильное время
- [ ] Позиции корректные
- [ ] Нет перекрытий
- [ ] Порядок спавна правильный

### Игровые состояния
- [ ] LOADING → INTRO
- [ ] INTRO → RUNNING
- [ ] RUNNING → PAUSED (туториал)
- [ ] RUNNING → END_WIN
- [ ] RUNNING → END_LOSE
- [ ] Переходы плавные

## Этап 5: Performance тестирование

### FPS
- [ ] 60 FPS на Desktop
- [ ] 60 FPS на мобильных (высокий уровень)
- [ ] 30+ FPS на слабых устройствах
- [ ] Нет просадок FPS

### Память
- [ ] Нет утечек памяти
- [ ] Память стабильна
- [ ] Очистка работает

### Загрузка
- [ ] Время загрузки < 3 секунд
- [ ] Прогресс загрузки отображается
- [ ] Нет блокировки UI
