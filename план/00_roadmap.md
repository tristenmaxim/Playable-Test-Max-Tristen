# Roadmap реализации

## Общий обзор

План разбит на 4 основных этапа с подзадачами. Каждый этап должен быть завершён и протестирован перед переходом к следующему.

---

## Этап 1: Фундамент (Неделя 1)

### Цель
Создать базовую структуру проекта и настроить основные системы.

### Задачи
- [ ] Настройка проекта (структура файлов, сборка)
- [ ] Инициализация PixiJS Application
- [ ] Система загрузки ассетов
- [ ] Базовые константы игры
- [ ] Система состояний игры
- [ ] Базовый игровой цикл (ticker)

### Результат
Работающее приложение с PixiJS, способное загружать ассеты и переключать состояния.

---

## Этап 2: Игровые системы (Неделя 2-3)

### Цель
Реализовать все игровые сущности и системы рендеринга.

### Задачи
- [ ] ParallaxBackground (фон с декорациями)
- [ ] Player (игрок с анимациями и физикой)
- [ ] Enemy (враги)
- [ ] Obstacle (препятствия)
- [ ] Collectible (коллекции)
- [ ] FinishLine (финишная линия)
- [ ] Система спавна сущностей
- [ ] Система коллизий (AABB)
- [ ] Физика прыжков

### Результат
Полностью функциональная игра с движущимися сущностями, коллизиями и физикой.

---

## Этап 3: UI и логика (Неделя 4)

### Цель
Добавить UI элементы и завершить игровую логику.

### Задачи
- [ ] HP Display (отображение здоровья)
- [ ] Score Display (отображение счёта)
- [ ] Tutorial Overlay (обучающий оверлей)
- [ ] End Overlay (экран победы)
- [ ] Fail Overlay (экран поражения)
- [ ] CTA Button (кнопка Call-To-Action)
- [ ] Система обработки ввода (touch/mouse)
- [ ] GameController (управление игрой)
- [ ] Система событий
- [ ] ConfettiEmitter (конфетти при победе)

### Результат
Полная игра с UI, обработкой ввода и всеми игровыми состояниями.

---

## Этап 4: Полировка (Неделя 5)

### Цель
Оптимизация, тестирование и финальная полировка.

### Задачи
- [ ] Оптимизация размера бандла (< 5MB)
- [ ] Оптимизация производительности
- [ ] Responsive design (разные размеры экрана)
- [ ] Тестирование на разных устройствах
- [ ] Исправление багов
- [ ] Финальная полировка анимаций
- [ ] Документация кода

### Результат
Оптимизированная, протестированная игра готовая к деплою.

## Критические зависимости

### Этап 1 → Этап 2
- Без системы загрузки ассетов нельзя загрузить спрайты
- Без игрового цикла нельзя обновлять сущности

### Этап 2 → Этап 3
- Без игровых сущностей нельзя создать UI для отображения состояния
- Без GameController нельзя управлять игрой

### Этап 3 → Этап 4
- Без полной функциональности нельзя оптимизировать

---

## Риски и митигация

### Риск 1: Размер бандла > 5MB
**Митигация**: 
- Оптимизация ассетов (WebP, сжатие)
- Минификация кода
- Удаление неиспользуемых ассетов

### Риск 2: Низкая производительность на мобильных
**Митигация**:
- Object pooling
- Оптимизация рендеринга
- Ограничение resolution до 2

### Риск 3: Проблемы с ассетами
**Митигация**:
- Ранняя проверка всех ассетов
- Fallback текстуры
- Проверка форматов
